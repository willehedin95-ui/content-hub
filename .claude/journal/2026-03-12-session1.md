## Decisions made
- **AbortController pattern**: All cancel implementations use `AbortController` with signal propagation to `fetch()`. The `AbortError` is caught and silently ignored (no error shown to user). This is the standard web API approach and works cleanly with NDJSON streaming.
- **Tags as flat strings**: Tags are stored as a `string[]` in the database. We use lowercase normalization for deduplication and filtering but display as-is. No separate tags table — keeps it simple.
- **Suggested tags**: Hardcoded list of 10 suggested tags matching common advertising angles and content types. These appear as clickable chips below the tag input.
- **Image panel stays open**: Only removed auto-close for generation operations (translate, replace, smart-generate). Manual operations (upload, product bank pick, revert) still auto-close because those are explicit user actions where the result is immediately visible.

# Session: 2026-03-17 (session 2) — Builder Improvements

## What was done
- **Fix text editor for inline elements** — Added STRONG, EM, B, I, U, S, MARK, SMALL, SUB, SUP, CITE, Q, CODE, TD, TH, DT, DD, SUMMARY to `TEXT_TAGS` in TextEditorControl.tsx so rich text editor appears for these elements. Commit `3a1f6df`.
- **Object-fit + object-position controls** — Added object-fit selector (cover/contain/fill/none) and object-position control (3x3 grid + X/Y % inputs) to SizeControl.tsx. Fixes stretched images when replacing square images with landscape ones.
- **Fixed image selection UX** — Images now get the standard blue selection box like all other elements. Removed `stopPropagation()` from iframe click handler so image clicks bubble to normal selection. Changed `cc-image-click`/`cc-video-click` from auto-opening ImagePanel/VideoPanel to storing data in refs. Added explicit "Replace / Generate Image" and "Replace Video" buttons in ConfigTab. Removed old `data-cc-img-highlight`/`data-cc-media-highlight` CSS.
- **AI Edit sidekick** — New Replo-inspired AI editing feature in the builder's AI tab. Free-form instruction input with scope selector (element/section/page), quick actions (Shorten text, More urgent, Simpler language, Add social proof), undo support. New API route `/api/builder/ai-edit` using Claude Sonnet. Commit `c6530ac`.

## Decisions made
- **Refs instead of state for media data** — `lastImageDataRef`/`lastVideoDataRef` store clicked image/video metadata without triggering re-renders or auto-opening panels. The user explicitly opens the image/video editor via buttons.
- **Scope-based AI editing** — Element scope sends selected element's innerHTML, Section scope walks up DOM to find section/article/container, Page scope sends full body. This gives users control over how much the AI can modify.
- **Quick actions as predefined instructions** — Rather than building separate UIs for common edits, they're just pre-filled instruction strings sent to the same AI edit API. Easy to extend.

## Decisions made
- **Server-side tab routing for pages/products**: Used Next.js 15 `searchParams: Promise<>` pattern in server components to conditionally fetch data per tab — only loads what's needed
- **Client-side tabs for brainstorm**: Since brainstorm was already a client component with URL-based tabs, added hooks/learnings as additional client-side tabs
- **Extracted components vs inline**: Hooks and Learnings extracted into `src/components/brainstorm/` as headerless components (no page-level icon/title since brainstorm provides that context)
- **getSession() in middleware**: Accepted the tradeoff — `getSession()` reads from cookie (no server validation) but the actual security is enforced by RLS in Supabase anyway. Saves a network call per request.

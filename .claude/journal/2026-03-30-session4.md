## Decisions made
- **JSON for closeup/messy only, NOT medical**: Medical illustrations benefit from dense narrative prose with anatomical terminology — JSON would fragment the detail. Only photographic styles get JSON.
- **Feature flag approach**: Rather than a git-only revert, a boolean constant lets the user A/B test JSON vs plain text without code changes
- **JSON stringify in parseBriefs()**: The `ImageBrief.prompt` stays as `string` type. JSON objects are stringified at parse time, so all downstream code (generate-static-images, autopilot, iterate, re-roll) works without changes.
- **Competitor swipe JSON handling**: When Claude returns a JSON object prompt AND text overlays are needed, the overlay info is added as `TextOverlay`/`ProductDescription` keys to the JSON object before stringifying. Plain text prompts keep the old append-string behavior.

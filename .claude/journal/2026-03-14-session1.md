# Session: 2026-03-14 (Invoice Tracker — Manual Flow & Matching Fixes)

## What was done
- **Disabled auto-forwarding to Juni**: Changed `processInvoices()` to store matched invoices as "ready" status instead of auto-forwarding. User now manually triggers forwarding via UI buttons.
- **Added manual forward flow**: New `forwardLogToJuni(logId)` function downloads email from IMAP on demand and forwards to Juni. Per-row "Send" button + bulk "Send All to Juni" banner in UI.
- **Created forward-all endpoint**: `/api/invoices/forward-all` bulk forwards all "ready" logs for a period.
- **Cleaned up Shopify logs**: Deleted 68 junk logs (customer inquiries, PayPal notifications), 34 duplicates, reassigned 14 logs to correct services (Meta, Vercel, Lovable, etc.).
- **Fixed overly broad service matching rules**:
  - Klaviyo: `@stripe.com` → specific Klaviyo Stripe address
  - Vercel: subject `"receipt"` → subject `"Vercel"`
  - Meta: subject `["invoice","receipt","payment","billing"]` → subject `["Meta ads"]`
  - Added specific Stripe senders for Ron Image Ads, Well Copy, Lovable
  - Added subject pattern for Shelfless logistics invoices
- **Reset last_processed_uid to 0** for full rescan after rule fixes
- **Bumped scan timeout** from 45s to 55s to handle full rescans
- **Added "Ready" status badge** (indigo color with Send icon)

## Decisions made
- **Two-phase forwarding**: Scan → "ready" → manual send → "forwarded". This prevents sending garbage to Juni while rules are being refined.
- **First-match-wins ordering**: Services matched alphabetically, so broad patterns (like Shopify's `@shopify.com`) must be narrowed to avoid grabbing other services' emails.
- **Forwarded-from-hotmail emails** lose original sender — must use subject-based matching for personal subscriptions (Anthropic, Vercel, Meta, Thinkific).

## What was done

**Root cause investigation + six bug fixes (commit `47e8a04`):**

Investigated why the launchpad wasn't pushing and why /review was showing inconsistent data. Found six distinct bugs — not one quick fix, six root causes. All fixed and pushed.

1. **`finish-and-queue` bypassed `approveConceptAction`** (main launchpad culprit)
   - Route set `launchpad_priority` directly but never created `image_job_markets` or `concept_lifecycle` entries
   - `pipeline-push` cron queries concepts via markets → couldn't find them → launchpad silently stalled
   - Fix: now calls `approveConceptAction(jobId, "finish_and_queue")` first, which handles markets + lifecycle + LP auto-assign

2. **`rejectConceptAction` only set `archived_at`, not `status`**
   - Other code paths (e.g. manual archive buttons) set `status='archived'` without `archived_at`
   - `/review/pending` filters on `archived_at IS NULL`, so status-only archives still appeared in review, and archived_at-only rejects still appeared in other status-filtered views
   - Fix: now sets both fields atomically

3. **`/review/pending` filter was too permissive**
   - Query used `.neq("status", "draft")` which still allowed `completed`, `processing`, `failed`, `archived` concepts
   - Fix: changed to strict `.eq("status", "ready")`

4. **Image count mismatch (3 in /review vs 6 in concept page)**
   - `ReviewCard` sliced thumbnails to 3 but displayed no indication of extra images
   - Fix: shows up to 3 previews with a `+N` overflow badge on the last thumbnail when there are more

5. **Workspace context bug: Hydro13 concept opened in HappySleep workspace**
   - `/review` is cross-workspace, but clicking a concept used an `<a>` tag that didn't update the `ch-workspace` cookie
   - Fix: replaced `Link` with a button that sets `document.cookie = "ch-workspace=<slug>"` before `window.location.href = url`

6. **No recovery mechanism for stuck jobs**
   - When Vercel timed out mid-pipeline or an upstream call threw, concepts got stuck in `draft`/`processing` forever
   - Fix: added `reconcileStuckJobs()` as Step 0 of `pipeline-push` cron:
     - Resets `image_translations` stuck in `processing` for >2h → `pending`
     - Promotes stuck autopilot `draft` concepts >6h old with `source_images` → `ready`
     - Marks `image_jobs` stuck in `processing` >2h with no pending translations → `completed`

## Decisions made

- **Root cause over quick fix**: User explicitly demanded "vi får ju lösa problemet, inte göra quick fixes wtf?". Instead of manually resetting the stuck rows, I added `reconcileStuckJobs()` so the system recovers automatically next time Vercel times out. The manual DB cleanup was still done, but it's now a backup plan, not the fix.
- **Reconcile thresholds**: 2h for stuck processing translations (short enough to catch real issues fast, long enough to not interfere with legitimate in-progress work), 6h for stuck autopilot drafts (covers a full day's cron window).
- **Status authority**: Chose `status='ready'` as the source of truth for "ready to review" instead of `launchpad_priority IS NULL` alone, because draft concepts don't have LP either. Now /review has a single clear filter.
- **Workspace cookie set client-side**: Setting `ch-workspace` via `document.cookie` + hard navigation is the simplest fix without needing server-side redirects or a `?workspace=` query param. Works because the cookie is read server-side on the next request.

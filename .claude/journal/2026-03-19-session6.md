- **Processing poll fix** — After "Finish & Queue", the progress bar showed "0/0" because there was no polling for server-side `after()` pipeline progress. Added a 5-second polling interval when `job.status === "processing"` and the client isn't doing the processing itself.

## Decisions made
- **Removed preview modal click**: User said "we dont need a bigger preview anymore." Cards no longer open the ImagePreviewModal on click. The modal component is kept (still used elsewhere) but not triggered from the image grid.
- **Card grid ratios**: Used `aspect-[4/5]` for thumbnails to match the ad format — shows full image without cropping.
- **Processing poll vs client processing**: The new poll only runs when `proc.processing` is false (server-side pipeline). When the client is processing translations itself, it has its own watchdog timer.
- **Re-roll for competitor-swipe**: Re-roll reuses the SAME prompt with a different seed for competitor-swipe images. It does NOT apply Rule 11 text placement since the prompt is preserved. For native (no-text) images this is fine. For text images, a full re-swipe from scratch would be needed.

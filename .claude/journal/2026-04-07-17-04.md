### 4. Vercel deploy failed on `maxDuration = 800` (commit `565ad43` - revert)
Pushed everything, Vercel started building, came back **Error**.

```
Error: Builder returned invalid maxDuration value for Serverless Function
"api/cron/autopilot-concepts". Serverless Functions must have a maxDuration
between 1 and 300 for plan hobby.
```

**Root cause:** The `maxDuration = 800` set in commit `8b33087` (this morning) was based on a wrong assumption that the project was on Vercel Pro with fluid compute. Verified via team billing API - project is on **hobby plan**, hard cap 300s. The `8b33087` commit was sitting unpushed in the local repo and got deployed for the first time today as part of this push, exposing the bug.

**Verification:**
```bash
curl https://api.vercel.com/v2/teams/team_4bxkDOt3vGRX8rlUanMlXBhp \
  -H "Authorization: Bearer $VTOKEN"
# -> "plan": "hobby"
```

Even though `defaultResourceConfig.fluid: true` was set in the project, hobby plan caps function duration at 300s regardless.

**Fix:** Reverted `maxDuration` from 800 to 300 in `autopilot-concepts/route.ts`. Updated the comment block to reflect the new reality. Parallelization improvements from `8b33087` are still valid - 3 concepts × ~100s parallel image gen = ~300s, which is feasible. If a run gets killed mid-loop, the reconcile job in pipeline-push catches the partial state.

Build verified clean. Pushed. Deploy succeeded as `565ad43` ● Ready.

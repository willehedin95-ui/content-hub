## Decisions made
- **Overlay layout instead of route groups**: Used `fixed inset-0 z-50` pattern (proven in builder) rather than restructuring app into route groups. Zero root layout changes needed.
- **Shared approval-actions.ts**: Extracted DB operations into pure functions that take only `jobId` + `source`. Both Telegram and review page call same code. `after()` for translations stays in the route handlers (not in shared lib) since it's a Next.js API route feature.
- **Keep Telegram inline buttons**: For backward compat with existing messages. Added review link alongside buttons. Can remove buttons later once `/review` is proven.
- **Cross-workspace queries use service role**: No `getWorkspaceId()` — joins `workspaces` table for metadata. Auth still enforced by middleware (Supabase session cookie required).
- **Filter out incomplete items**: Draft concepts without images and video jobs without keyframe images are excluded — nothing visual to review.

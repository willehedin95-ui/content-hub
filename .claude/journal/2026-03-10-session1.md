## What was done
- **Replo/Figma-style sidebar redesign** — rewrote 3 core design controls:
  - `LayoutControl` — 3×3 alignment grid (replacing separate justify/align dropdowns), Packed/Spaced toggle, direction arrows, wrap toggle
  - `SpacingControl` — combined nested margin+padding box (amber outer "M", indigo inner "P", white "content" center)
  - `SizeControl` — Fill/Wrap/Fixed preset buttons for width, Hug/Fill/Fixed for height, collapsible Min/Max advanced section
- **DesignTab reorder** — new order: Size → Auto Layout → Spacing → Typography → Background → Border → Effects → Position. Dynamic section title: "Auto Layout" when flex/grid, "Layout" otherwise
- **Container-first hover and selection** — hover highlights parent container by default, Cmd+hover bypasses to deepest child, double-click drills one level deeper, Escape goes up one level or deselects
- **Figma-style hidden layers** — removed amber "X hidden" banner from layers panel, hidden layers remain in tree with grey text/icons and always-visible eye-off icon for toggling
- **Typography section moved** directly below text editor (pushed earlier as `1701a48`)
- **Hover highlighting** — CSS hover outlines + floating tag labels on canvas (pushed earlier as `1701a48`)

## Decisions made
- **Rejected resize handles and snap lines** — user said these are Figma free-canvas concepts, not applicable to stacked web page builders. Auto layout is the correct feature instead.
- **Container-first as default** — following Replo/Figma pattern: hovering selects top-level sections, not deepest children. Direct child access via Cmd+hover.
- **Combined spacing box** — Replo-style nested margin+padding replaces separate boxes. More compact, more intuitive.
- **Removed HV (horizontal/vertical) toggle** from spacing — not enough room in nested layout and rarely used.
- **Removed revealHidden banner** — Figma keeps hidden layers in the tree with eye icons, no need for a separate reveal mechanism.

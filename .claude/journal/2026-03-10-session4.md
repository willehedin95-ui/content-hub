- **Image generation fallback**: When source image is broken/CORS/404, GPT Vision fails gracefully and retries in text-only mode using surrounding text context

## Decisions made
- Two-phase generation: images first (parallel, fast) then videos (sequential with Kling polling) — keeps UX responsive for the fast stuff
- VIDEO/IMAGE toggle per video item rather than global setting — user can choose which videos to keep as video vs replace with static image
- Text-only fallback uses the same prompt quality as intentional text-only mode (not a degraded path)

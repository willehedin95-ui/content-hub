## Decisions made
- **Anti-panic as core principle**: The entire feature is designed to prevent the user's earlier panic (lowering budgets on a 7-day dip when 30-day was profitable). Multi-window display forces seeing the bigger picture.
- **Strategy engine as separate module**: Kept computation logic in `strategy-engine.ts` rather than inline in the API route — makes it testable and reusable.
- **Non-fatal strategy computation**: Wrapped in try/catch so existing morning brief works even if strategy engine has issues (e.g., empty tables).
- **Batch kill via re-fetch**: Telegram callback handler re-fetches strategy data rather than storing ad set IDs (Telegram callback_data has 64-byte limit).
- **Budget cooldown detection**: Uses `campaign_budget_snapshots` table to detect when user changed budget, then applies 3-day cooldown before recommending further changes.

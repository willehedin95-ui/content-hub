# Session: 2026-03-11 ~13:30

## What was done
- Rewrote the builder's "Generate for HappySleep" API (`/api/builder/generate-image/route.ts`) to use Claude Vision structured extraction instead of GPT-4o
- The new approach mirrors the Assets Image Swiper logic: Claude Vision extracts structured JSON (camera perspective, composition, subjects, colors, lighting, style), then swaps the competitor product with HappySleep data, then passes the full JSON to Nano Banana
- Key fix: surrounding text (headlines/paragraphs) is now passed as thematic context only — explicitly told "do NOT include any of this text in the image". Previously GPT-4o was copying section headlines directly into generated images
- Product hero images now passed to Nano Banana as references (builder wasn't doing this before)
- Committed and pushed as `b85813c`, deployed to Vercel

## Decisions made
- **Claude Vision over GPT-4o**: The Assets Image Swiper already proved Claude Vision's structured extraction produces much better replications. Using the same approach in the builder ensures consistency and better results.
- **Surrounding text as theme hint only**: Instead of making it the "PRIMARY guide", it's now a brief theme summary appended to the instruction field. This prevents headline text from being burned into images.
- **Kept same API response format**: `{ imageUrl, prompt, analysis }` — no client-side changes needed in ImagePanel.tsx
- **Text-only mode preserved**: When there's no competitor image (e.g., replacing a video placeholder), a separate Claude prompt generates structured JSON from scratch based on section theme

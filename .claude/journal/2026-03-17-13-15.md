## Decisions made
- **Used `after()` from Next.js 15** instead of fire-and-forget fetch or a new cron — gives immediate response to user while translations process in background, and the serverless function stays alive up to maxDuration (300s). Much simpler than a separate cron or endpoint.
- **Shared library instead of HTTP calls** — both Hub and Telegram handlers use the same `triggerAutopilotTranslations()` function directly, avoiding cookie/workspace dependency issues that HTTP endpoint calls would have.
- **Parallel Kie AI calls** — all 4:5 translations fire in parallel (~30s total instead of ~270s sequential). 9:16 runs after 4:5 completes since it depends on the 4:5 output for outpainting.
- **Bumped Telegram webhook maxDuration to 300** — was 120, but with translations running via `after()`, the function needs to stay alive long enough for ~9 parallel Kie AI calls + 9 outpainting calls.

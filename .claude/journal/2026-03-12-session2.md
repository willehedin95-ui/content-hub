## Decisions made
- **Ad-level testing instead of router-based**: Mark's advice — never split test on the first page where Meta sends traffic (crawler detects code changes, treats as cloaking)
- **Not using Meta's built-in Experiments**: Buggy, duration-locked, constrains algorithm, poorly documented API
- **Kept `ab_tests` table in DB**: Not dropping it — historical data preserved, just stopped writing to it
- **Page B creative reuses image hashes**: Meta stores uploaded images by hash, so page B ad set reuses the same images without re-uploading
- **Statistical significance via Z-test on purchase CVR**: Minimum 30 clicks per variant before testing
- **`shopify.ts` dead code left in place**: `getConversionsForTest()` no longer imported anywhere but leaving it in the Shopify module is harmless

# Session: 2026-03-19 Session 4 — Concept Detail Redesign + Meta Placement Fix

## What was done
- **Concept detail page redesign** — Replaced 3-step wizard (Images → Ad Copy → Preview & Push) with clean collapsible sections. Status-driven CTA adapts to concept lifecycle: generating → processing → needs_copy → ready → on_launchpad → live. Overflow menu for secondary actions (iterate, refresh, delete). Tags, CASH DNA, lineage moved to collapsed Details section at bottom.
- **4:5/9:16 ratio toggle** — Pill switcher next to language tabs in ConceptImagesStep. Changes thumbnail aspect ratio and URL source to preview outpainted 9:16 images. Translation status badges filtered by selected ratio.
- **Landing page standalone card** — Moved landing page selection out of Ad Copy section into its own card between images and ad copy. Cleaner separation of concerns.
- **Meta image placement routing fix** — User discovered 4:5 images appearing in Stories/Reels placements via Meta Ads Manager screenshots. Root cause: push code dumped all images into one big DCO creative with no `asset_customization_rules`, relying on Meta's "Adapt to Placement" which doesn't work reliably. Fix: per-image-pair creatives with explicit `asset_customization_rules` routing 4:5→feed placements and 9:16→stories/reels placements. `FEED_STORIES_RULES` constant with 7 rules covering all Meta/IG/AN/Messenger placements.
- **Campaign builder push fix** — Same `asset_customization_rules` fix applied to `/api/meta/campaigns/[id]/push/route.ts` which had the old pattern (uploaded 9:16 but never used it in creative).
- **Bug fixes** — "Finish & Queue" CTA now correctly shows "Add to Launch Pad" when concept is fully ready. Processing banner no longer shows "Processing 18/18" when everything is done (was caused by `finishQueue.started` never resetting).

## Decisions made
- **Per-image-pair creatives instead of one big DCO** — Each image pair (4:5 + 9:16) gets its own creative with `asset_customization_rules`. Titles limited to 1 when rules are present (Meta constraint subcode 1885878). Without 9:16, falls back to single creative with all images (no rules).
- **Status check order matters** — `computeConceptStatus()` checks "fully done" BEFORE "processing" because `finishQueue.started` stays true after pipeline completes and never resets.
- **FEED_STORIES_RULES exported** — Shared between meta-push.ts and campaign builder route to avoid duplication.

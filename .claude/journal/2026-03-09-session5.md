# Session: 2026-03-09 (Session 5) — Page Builder UX Overhaul

## What was done
- **Delete without confirm**: Removed confirm dialog on element deletion — undo via Ctrl+Z is sufficient protection
- **Backspace/Delete key**: Added keyboard shortcut to delete selected elements (both main window and iframe), with safeguards when typing in inputs
- **Font size bug fixed**: Design controls (font size, weight, color, etc.) now properly update when switching between elements. Root cause: `hasSelectedEl` stayed `true` when clicking from one element to another, so useEffects never re-fired. Fix: increment `layersRefreshKey` on every selection change.
- **Freshchat/third-party script stripping**: `handleIframeLoad()` now removes all `<script>` tags (except editor-injected) and foreign iframes. This prevents imported chat widgets, analytics, etc. from running live in the editor and being unselectable/undeletable.
- **Image/video selection**: Removed the skip for IMG/VIDEO in click handler. PostMessage handler now also calls `selectElementInIframe()` so clicked images highlight in canvas + layers panel.
- **Layers panel redesign**: Complete rewrite for Figma-like experience — shows DIVs/containers, tag icons, auto-expands to selected element, auto-scrolls selected into view, smarter labels using class names, deeper tree (8 levels), flattens trivial single-child wrappers.
- **AI image generation panel fix**: Panel no longer closes immediately when generating — stays open with a spinner ("Generating..."/"Translating..."), only closes on success. Errors shown inline.
- **Rich text editor**: New `TextEditorControl` in Design tab with Bold/Italic/Underline/Strikethrough/Link/RemoveFormat toolbar. Uses contentEditable in the panel with two-way sync to canvas. Removed old plain textarea from Config tab. Removed inline contentEditable behavior from preview script — text elements now select normally like any other element.

## Decisions made
- **Rich text in Design tab, not Config tab**: User explicitly asked for this since switching tabs after selecting text is annoying. Text editor appears at the top of Design tab for immediate access.
- **No inline editing in canvas**: Removed `contentEditable` behavior from the preview script entirely. All text editing goes through the right panel editor. This gives cleaner selection behavior (text elements highlight like any other element).
- **Strip scripts on editor load, not on save**: Scripts are removed from the iframe DOM during `handleIframeLoad()`. Since `extractCleanHtml()` reads from the iframe DOM, stripped scripts won't be in saved HTML either. This is the desired behavior — swiped pages shouldn't keep original tracking/chat scripts.
- **Reuse `layersRefreshKey` for selection change**: Instead of adding a new `selectionKey` state (which caused TypeScript linter hook to revert changes due to incremental edit issues), reused the existing `layersRefreshKey` counter. It's already a dependency in all design control useEffects.
